Bus Booking System Summary
==========================

ระบบจัดการการจองตั๋วรถบัส พัฒนาด้วยเทคโนโลยี **Node.js** และ **Express** เน้นการจัดการข้อมูลหลังบ้าน (Backend Management) และการเชื่อมต่อฐานข้อมูลอย่างเป็นระบบ

Tech Stack
----------

*   **Backend:** Express.js
    
*   **Database:** MySQL
    
*   **Frontend:** EJS Template Engine
    
*   **Styling:** Tailwind CSS & Custom CSS
    
*   **Development Tools:** Nodemon
    

Database Schema
------------------------------------

ระบบประกอบด้วย 4 ตารางหลักที่มีความสัมพันธ์กัน ดังนี้:

**TableDescriptionKey FieldsMember**ข้อมูลสมาชิกชื่อ, เบอร์โทร, จำนวนครั้งที่เข้าใช้บริการ (travel\_count)**Bus**ข้อมูลรถบัสทะเบียนรถ, ประเภทรถ, จำนวนที่นั่ง**Schedule**ตารางเวลาสถานที่ต้นทาง, ปลายทาง, เวลาออกรถ**Ticket**ข้อมูลการจองวันที่จอง, เลขที่นั่ง, ID สมาชิก, ชื่อเจ้าหน้าที่

Main Features
---------------------------

### 1\. หน้าหลัก

*   ศูนย์รวมเมนูนำทาง (Navigation Hub) เชื่อมโยงไปยังระบบจัดการส่วนต่าง ๆ
    

### 2\. การจัดการข้อมูลพื้นฐาน

*   **Member:** ระบบเพิ่มสมาชิกใหม่เพื่อเก็บประวัติการเดินทาง
    
*   **Bus:** บันทึกรายละเอียดรถบัสแต่ละคันและประเภทรถ
    
*   **Schedule:** กำหนดเส้นทางเดินรถและเวลาให้บริการ
    

### 3\. ระบบจองตั๋ว

*   ทำการออกตั๋วโดยระบุที่นั่งและสมาชิก
    
*   **Auto-Update:** เมื่อมีการจองตั๋ว ระบบจะทำการเพิ่มจำนวนครั้งการใช้บริการในตาราง Member ให้โดยอัตโนมัติ
    

### 4\. การค้นหาและประวัติ

*   **Search:** ค้นหาข้อมูลตั๋วแบบละเอียดผ่าน "หมายเลขตั๋ว" หรือ "ชื่อสมาชิก"
    
*   **History:** ตรวจสอบประวัติเส้นทางย้อนหลังตามหมายเลขทะเบียนรถบัสที่ระบุ
    

Workflow
---------------------------------

1.  **Entry:** ผู้ใช้เข้าสู่ระบบผ่านหน้า **Home**
    
2.  **Data Input:** เจ้าหน้าที่กรอกข้อมูลพื้นฐาน (สมาชิก, รถบัส, ตารางเวลา) ลงสู่ฐานข้อมูล MySQL
    
3.  **Booking:** เมื่อมีการจองตั๋ว ระบบจะเชื่อมโยงข้อมูลจากตาราง Member และ Schedule เข้าด้วยกัน
    
4.  **Process:** ระบบตรวจสอบและบันทึกข้อมูลลงตาราง Ticket
    
5.  **Trigger:** ระบบทำการ UPDATE จำนวนการใช้งานของสมาชิกในฐานข้อมูลทันที
    
6.  **Output:** สามารถเรียกดูรายงานผ่านหน้า Search และ History ได้ตลอดเวลา
